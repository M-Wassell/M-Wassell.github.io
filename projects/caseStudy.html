NHS Acronym Management System

Project Overview

A .NET 8 MVC web application developed to help NHS staff search, manage, and understand service acronyms used across different departments.

The Problem

The NHS uses a large number of acronyms across different departments and services. Staff working across clinical areas may encounter unfamiliar acronyms, which can slow down communication and create confusion.

There was no centralised system allowing staff to quickly search or contribute commonly used acronyms.

The Solution

I developed an internal web application using ASP.NET .NET 8 MVC that allows NHS staff to:

  • Search for acronyms used across services
  • Submit new acronyms for inclusion
  • View a shared repository of commonly used terms

The system validates user input and ensures appropriate content is submitted before storing entries in a shared database accessible by staff.

Key Features

  • Acronym search functionality accessible to all staff
  • Secure submission form for new acronym entries
  • Client-side validation using JavaScript
  • Server-side validation using ASP.NET model validation
  • SQL database storage for acronym records
  • Profanity filtering system to prevent inappropriate submissions

Technical Architecture

The application follows a standard MVC architecture:

  • Model – Handles acronym data and validation rules
  • View – Razor views used to display search results and input forms
  • Controller – Manages user input, validation, and database interaction

Data is stored in a SQL Server database, with validation layers implemented to ensure both security and data integrity.

Input Validation & Data Integrity

To ensure the system remained appropriate for a professional environment, I implemented:

  • Client-side validation using JavaScript
  • Server-side validation using ASP.NET model validation
  • A profanity filtering system

The profanity filter checks user input against a curated list of approximately 800 restricted words stored in a SQL database, preventing inappropriate submissions.

Technologies Used

  • .NET 8
  • ASP.NET MVC
  • C#
  • SQL Server
  • JavaScript
  • HTML / CSS
  • Visual Studio 2022

Challenges

One of the main challenges was ensuring user-generated content remained appropriate while still allowing staff to contribute new entries.

To address this, I implemented both client-side and server-side validation along with a database-driven profanity filter.

Impact

The system provides staff with a centralised resource for understanding service acronyms used across the organisation.

By allowing staff to contribute new entries, the application ensures the acronym database can grow over time as new services and terminology are introduced.

What I Learned

  • Building production applications using .NET 8 MVC
  • Implementing secure input validation
  • Designing database-driven validation systems
  • Developing software for real operational users
